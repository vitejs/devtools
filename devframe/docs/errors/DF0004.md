# DF0004: Terminal Session Already Registered

## Message

> Terminal session with id "`{id}`" already registered

## Cause

`DevToolsTerminalHost.register()` already has a session under that id.

## Fix

Use a unique session id or call `unregister(id)` first.

## Source

- [`packages/devframe/src/node/host-terminals.ts`](https://github.com/vitejs/devtools/blob/main/devframe/packages/devframe/src/node/host-terminals.ts) — `DevToolsTerminalHost.register()` (and the internal terminal-creation path) throw `DF0004` when a session id is already in use.
- [`packages/devframe/src/node/diagnostics.ts`](https://github.com/vitejs/devtools/blob/main/devframe/packages/devframe/src/node/diagnostics.ts) — defines `DF0004`.
